Cloudflare Bypass Root Cause Diagnosis

* Identified 4 specific flaws in the SeleniumBase Layer 3 implementation
  causing Cloudflare Turnstile bypass failure

  * headless=True leaks bot signals: empty plugins, SwiftShader WebGL
    renderer, zero screen dimensions -- all detectable by CF Bot Management

  * uc_gui_click_captcha() is wrong method for Turnstile -- it targets
    reCAPTCHA v2 image grids, Turnstile verifies passively with no click

  * 3-second sleep insufficient -- Turnstile fingerprinting takes 8-15
    seconds on suspicious sessions before resolving

  * Fresh Chrome instance per article -- no cookie persistence, zero CF
    trust score on every request

* Confirmed root cause is architectural: Blockworks uses Cloudflare Bot
  Management (paid enterprise tier) which evaluates IP reputation and TLS
  fingerprint at network level -- no browser automation tool defeats this

CHALLENGES RESOLVED

Blockworks returning 140 chars of content on every article fetch

* Root cause:
  Cloudflare Bot Management operates at TLS and IP reputation level --
  no headless browser tool can pass it from a flagged IP regardless of
  UC Mode or stealth patches

* Fix:
  Switched to blockworks.com/api/posts JSON endpoint which is
  completely unprotected -- returns full article HTML content in one
  request with no Cloudflare involvement

Date parsing failure on Blockworks API response

* Root cause:
  API returns dates in RFC 2822 format (Wed, 07 Jan 2026 14:00:00 GMT)
  which datetime.fromisoformat() cannot parse

* Fix:
  Added two-stage parser in _parse_date(): ISO 8601 first, then
  RFC 2822 with GMT replaced by +0000 for strptime compatibility

CHALLENGES RESOLVED

Playwright networkidle timeout on Cloudflare pages

* Root cause:
  Cloudflare challenge page fires continuous background XHR requests so
  networkidle (no network for 500ms) never triggers, causing 30s timeout

* Fix:
  Changed wait_until to "load" and increased timeout to 60s

Both headless tools fail against Blockworks Bot Management

* Root cause:
  Blockworks uses Cloudflare Bot Management which fingerprints at the TLS
  and network level -- in-page stealth patches cannot override this tier

* Fix:
  Kept SeleniumBase as Layer 3 for sites with basic CF protection;
  Blockworks falls back to RSS content via the 300-char minimum guard
